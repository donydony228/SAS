NYU x SAS Cortex: 慈善募款預測與營運優化專案📌 專案背景本專案為 NYU x SAS Cortex Data Analytics Challenge 的競賽實作。目標是協助一家擁有百萬會員的非營利慈善機構（Foundation），透過數據科學模型與商業決策分析，優化其直接聯繫（Direct Contact）行銷活動的成效。🎯 核心目標最大化營運盈餘 (Maximize Operating Surplus)。在募款過程中，每一筆聯繫（電話或郵件）皆有成本。本專案的目標不只是「找出會捐款的人」，而是「找出聯繫後能產生最大淨利潤的人」。$$Operating\ Surplus = \sum(\text{Actual Donation Amount}) - \sum(\text{Contact Cost})$$🛠 解決方案與方法論本專案採用 Kaggle 式的快速迭代開發流，結合預測建模與商業邊際分析：1. 雙模型預測架構 (Two-stage Modeling)為了應對數據中嚴重的「零值偏誤（Zero-inflation）」，我們不直接預測金額，而是拆解為兩個階段：分類模型 (Classification)：使用 Gradient Boosting / Random Forest 預測會員的捐款機率 ($P$)。回歸模型 (Regression)：針對歷史捐款者，預測其潛在捐款金額 ($E$)。期望價值計算：$EV = P \times E$。2. 邊際貢獻決策 (Marginal Value Optimization)我們並非聯繫所有預測會捐款的人，而是基於以下四象限邏輯進行篩選：目標群體 (Persuadables)：聯繫後的期望收益 > 聯繫成本 $\rightarrow$ 納入名單。超級鐵粉 (Sure Things)：識別出不聯繫也會自發捐款的人 $\rightarrow$ 排除聯繫，以節省成本並最大化淨利。無感者 (Lost Causes)：預期收益過低 $\rightarrow$ 排除聯繫。3. 模型解釋性 (XAI)利用 SHAP (SHapley Additive exPlanations) 與 變數重要性 (Variable Importance) 分析關鍵因子：分析哪些特徵影響「捐款意願」（如：Recency 距今時間）。分析哪些特徵影響「捐款深度」（如：Salary 年薪）。🚀 執行流程EDA (探索性資料分析)：分析變數分佈，確認零值膨脹程度與階梯式成本結構。模型訓練 (SAS Viya)：訓練並集成多組分類與回歸模型。利潤模擬：在 Excel 中導入模型權重，計算每個 ID 的淨預期盈餘。動態篩選：根據 SAS 提供的 Cost Schedule，動態調整篩選門檻，確保在成本跳漲點前達到利潤極大化。迭代優化：根據排行榜 (Leaderboard) 回饋，持續修正模型參數。📈 預期成果產出精確的潛在捐款者聯繫名單 (ID List)。提供可解釋的模型洞察，協助機構理解核心捐款者的行為特徵。在受限的聯繫成本下，實現最高額的募款淨利潤。
