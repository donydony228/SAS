# X-Learner 完整說明

## 一、為什麼需要 X-Learner？

### 問題背景

我們想做一件事：**決定要聯繫哪些人，才能讓募款活動的淨利潤最大。**

直覺上，最好的策略是「只聯繫那些因為被聯繫而多捐款的人」。但問題是：

> 我們只知道每個人「被聯繫後的結果」或「沒被聯繫的結果」，**兩者不可能同時觀察到**。

這在因果推論裡叫做「反事實問題」（Counterfactual Problem）。

---

### 訓練資料的現實

R2_TRAIN 裡，100K 人分成兩群：

| 群體 | 人數 | 特點 |
|------|------|------|
| Contact=0（未聯繫）| 89,886 人 | 多，但沒有被聯繫過 |
| Contact=1（已聯繫）| 10,114 人 | 少，但有被聯繫的結果 |

問題：**Contact=1 只有 10K 人，用這 10K 人訓練「聯繫後行為模型」，樣本太少，很容易過擬合。**

我們之前試過的 T-Learner 就因此失敗：M1 的 Holdout AUC 只有 0.59，幾乎等於亂猜。

---

## 二、X-Learner 的核心想法

**借力打力：用 89K 人的知識，來幫助理解 10K 人的反應。**

具體做法：對每個「被聯繫的人」，問一個問題：

> 「如果他沒有被聯繫，他會捐多少？」

我們沒有這個答案，但我們可以用「89K 個沒被聯繫的人」訓練出來的模型去估計。這個估計比直接用 10K 人訓練穩定得多。

---

## 三、四個步驟詳解

### Step 1：訓練兩個基礎模型

把資料依照 Contact 分成兩群，各自訓練一個模型：

**M0（基準模型）**
- 訓練資料：Contact=0 的 71,872 人（80% train split）
- 特徵：人口資料 + 歷史捐款行為（共 19 個，**不含 Contact**）
- 預測目標：`GaveThisYear`（0 或 1）
- 代表意義：「在沒有任何聯繫的情況下，這個人有多高機率會捐款？」
- 訓練結果：Holdout AUC = 0.6615（用 89K 大樣本訓練，相對穩定）

**M1（聯繫後模型）**
- 訓練資料：Contact=1 的 8,128 人（80% train split）
- 特徵：同上 19 個特徵（**不含 Contact**）
- 預測目標：`GaveThisYear`（0 或 1）
- 代表意義：「在被聯繫的情況下，這個人有多高機率會捐款？」
- 訓練結果：Holdout AUC = 0.5901（樣本少，不穩定 — 這正是 X-Learner 要解決的）

---

### Step 2：計算偽目標（Pseudo Outcome）

這是 X-Learner 最關鍵的步驟。

**對被聯繫的人（Contact=1）：計算 D1**

```
D1 = 實際捐款結果（Y1）- M0 預測的「沒被聯繫時」捐款機率
```

直白解釋：
- M0 說：「假設這個人沒被聯繫，他有 20% 機率捐款」
- 他實際被聯繫後，捐款了（Y1 = 1）
- 所以 D1 = 1 - 0.20 = **+0.80**（被聯繫讓他「超預期」捐款了，是強力 Persuadable）

| D1 的值 | 含義 |
|---------|------|
| D1 > 0 | 被聯繫讓他比預期捐更多 → Persuadable 訊號 |
| D1 ≈ 0 | 聯繫效果中性 → Lost Cause |
| D1 < 0 | 被聯繫後反而比預期少捐 → Sleeping Dog 訊號 |

**為什麼這樣計算有意義？**
M0 是用 89K 人訓練的穩定模型，能給出可靠的「反事實基準」。D1 不是直接用 10K 人算出來的，所以不受樣本少的影響。

---

**對沒被聯繫的人（Contact=0）：計算 D0**

```
D0 = M1 預測的「被聯繫時」捐款機率 - 實際捐款結果（Y0）
```

直白解釋：
- M1 說：「假設聯繫這個人，他有 40% 機率捐款」
- 他實際沒被聯繫，也沒捐款（Y0 = 0）
- D0 = 0.40 - 0 = **+0.40**（如果聯繫他，預期比現在多 40% 機率捐款）

---

### Step 3：訓練 CATE 估計器

Step 2 算完後，我們手上有：
- 10K 個「被聯繫的人」各自的 D1
- 90K 個「沒被聯繫的人」各自的 D0

但評分時要對 **100K 個全新的人** 估計 Uplift。所以需要訓練兩個「Uplift 預測器」：

**τ1（被聯繫群的 Uplift 預測器）**
- 訓練資料：Contact=1 的 8,128 人
- 特徵：同上 19 個特徵
- 預測目標：**D1**（不是 0/1 的二元標籤，而是連續的 Uplift 值）
- 意義：給一個新的人，從「被聯繫者的角度」估計他的 Uplift 會是多少
- 訓練結果：Train R² = 0.3139（比之前的回歸模型 R²≈0 好很多，因為 D1 包含了 M0 的反事實資訊）

**τ0（未聯繫群的 Uplift 預測器）**
- 訓練資料：Contact=0 的 71,872 人
- 特徵：同上 19 個特徵
- 預測目標：**D0**
- 意義：給一個新的人，從「未被聯繫者的角度」估計他的 Uplift 會是多少
- 訓練結果：Train R² = 0.2450

---

### Step 4：傾向評分加權合併

現在有兩個 Uplift 估計，哪個更可信？

問題：Contact=1 的 10K 人**不是隨機選的**。歷史上有捐款記錄、比較活躍的人，本來就更容易被選入「聯繫名單」。這造成兩群人的特徵分布不同（選擇偏差）。

解法：先訓練一個「**傾向評分模型**」g(x)：

**Propensity Score 模型**
- 訓練資料：全量 80,000 人（train split）
- 特徵：同上 19 個特徵
- 預測目標：`Contact`（0 或 1）
- 代表意義：「根據這個人的特徵，他本來有多高機率被選入聯繫名單？」
- 訓練結果：Train AUC = 0.7210（Contact 確實非隨機分配，PS 加權有意義）

然後用 g(x) 決定加權方式：

```
最終 CATE = g(x) × τ0(x) + (1 - g(x)) × τ1(x)
```

| g(x) 的值 | 代表 | 加權結果 |
|-----------|------|---------|
| g(x) 高（例如 0.8）| 此人特徵很像「容易被聯繫的人」| 多信 τ0（control 群估計更中立）|
| g(x) 低（例如 0.1）| 此人特徵很像「不常被聯繫的人」| 多信 τ1（treatment 群估計更相關）|

## 四、所有模型一覽

| 模型 | 訓練資料 | 特徵數 | 預測目標 | 演算法 |
|------|---------|--------|---------|--------|
| M0 | Contact=0，71,872 人 | 19 | GaveThisYear（0/1）| GBM（Optuna 最佳參數）|
| M1 | Contact=1，8,128 人 | 19 | GaveThisYear（0/1）| GBM |
| τ0 | Contact=0，71,872 人 | 19 | D0（連續值）| GBM |
| τ1 | Contact=1，8,128 人 | 19 | D1（連續值）| GBM |
| Propensity | 全量，80,000 人 | 19 | Contact（0/1）| GBM |

超參數（全部相同，來自 Optuna 搜尋）：
- n_estimators=375, max_depth=6, learning_rate=0.0129
- subsample=0.604, min_samples_leaf=45

## 六、評分流程

訓練完 5 個模型後，對 100K 個 SCOREDATA 的陌生人進行評分：

```
1. X = preprocess(SCOREDATA)          ← 計算 19 個特徵
2. g  = Propensity.predict(X)          ← 每個人被聯繫的傾向
3. t1 = τ1.predict(X)                  ← 從「被聯繫視角」的 Uplift
4. t0 = τ0.predict(X)                  ← 從「未聯繫視角」的 Uplift
5. CATE = g × t0 + (1 - g) × t1       ← 加權合併
6. uplift_dollar = CATE × $65.21       ← 乘以均值，轉換成金額單位
7. 若 uplift_dollar > 成本 → 放入聯繫名單
```

## 七、結果

| 指標 | 數值 |
|------|------|
| 驗證集策略效率 | **84.9%**（GBM Single Model 為 70.3%）|
| SCOREDATA Persuadables | **60,000 人** |
| 預估成本 | $300,000（60K × $5，剛好在跳漲門檻）|
| 排行榜 Surplus | **$8,718,670**（最終最佳）|

## 八、X-Learner vs Level 2 Single Model

| 面向 | Level 2 Single Model | X-Learner |
|------|---------------------|-----------|
| **Contact 的處理方式** | 當成一個 feature，模型統一學習 Contact 的全局效應 | 透過分群 + 反事實估計，個別化計算每人 Uplift |
| **問的問題** | 「有 Contact 時，這個人的捐款率是多少？」 | 「**因為 Contact**，這個人多捐了多少？」 |
| **資料不平衡問題** | Contact=1 只佔 10%，訊號被 90% 稀釋 | 大群（90K）的知識被借來幫小群（10K）|
| **模型數量** | 2（分類器 + 回歸器）| 5（M0+M1+τ0+τ1+Propensity）|
| **個別化程度** | 低（所有人的 Contact 效應來自同一模型）| 高（每個人有自己的 D1/D0 → τ 估計）|
| **驗證效率** | 70.3% | **84.9%** |
| **排行榜 Surplus** | $8,688,900 | **$8,718,670** |
| **差異** | — | **+$29,770（+0.34%）** |

### Uplift 計算方式的根本差異

**Level 2 Single Model：**
```
同一個模型，預測兩次，取差值

模型輸入 [x, Contact=1] → P1
模型輸入 [x, Contact=0] → P0
Uplift = P1 - P0
```
Contact 是模型的一個普通 feature。模型學到的是「Contact=1 的人捐款率比 Contact=0 的人高 22%」這個**全局規律**，套用在所有人身上的方式大致相同。

**X-Learner：**
```
不用 Contact 當 feature，改用反事實差值直接估計每個人的個別效應

D1 = 這個人實際被聯繫的結果 − 預估沒被聯繫的結果（用 M0 算）
D0 = 預估被聯繫的結果（用 M1 算）− 這個人實際沒被聯繫的結果
CATE = 用 D1/D0 訓練出的 Uplift 估計器
```
每個人的 D1/D0 是基於他自己的情況算出來的，不是全局套用。

---

### 具體對比：同一個人，兩種計算

假設「55 歲女性，AmtLastYear=$0，NbActivities=2」：

**Level 2 Single Model：**
```
1. 把她塞進模型，Contact=1 → 預測 P1 = 0.35（35% 捐款率）
2. 把她塞進模型，Contact=0 → 預測 P0 = 0.13（13% 捐款率）
3. Uplift_prob = 0.35 - 0.13 = 0.22
4. Uplift_dollar = 0.22 × E_amount_model  （模型同時預測捐款金額）
```

**X-Learner：**
```
1. M0（89K 控制群）預測她「沒被聯繫時」捐款率 = 0.15
2. 她實際被聯繫了，結果有捐款（Y1 = 1）
   → D1 = 1 - 0.15 = +0.85（比沒聯繫的預期多了 85%）
3. τ1 學習：這個特徵組合的人，被聯繫後 CATE ≈ 0.18
4. 最終 CATE = 0.18（加入 Propensity Score 加權後）
```

**轉換成金額並做決策：**

X-Learner 用**捐款者全體均值**（$67.25）把機率 Uplift 轉換成金額：
```
uplift_dollar = CATE × $67.25
             = 0.18 × $67.25
             = $12.11  >  成本 $5 → 列入名單（Persuadable）
```

| 步驟 | Level 2 | X-Learner |
|------|---------|-----------|
| 分類模型數 | 1 個（Contact 是 feature）| 2 個（M0 + M1 分群訓練）|
| Uplift 估計 | 同一模型預測兩次取差 | D1/D0 反事實 → τ 估計器 |
| 金額估計 | 回歸模型直接預測 E(amt) | 固定用捐款者均值 $67.25 |
| 最終 Uplift | P_diff × E_amt | CATE × $67.25 |

---

### 為什麼用均值 $67.25 而非截尾均值？

捐款者金額分佈極度右偏（中位數 $25，均值 $67，最大 $10,000）。我們做過截尾均值的實驗：

| 截尾點 | 均值 | 選出人數 | 驗證 Surplus | 效率 |
|--------|------|---------|------------|------|
| **不截尾（採用）** | **$67.25** | **14,878** | **$147,240** | **84.9%** |
| 去掉 top 1%（>$750） | $49.25 | 13,910 | $139,040 | 80.2% |
| 去掉 top 5%（>$200） | $36.46 | 12,537 | $126,520 | 73.0% |
| 去掉 top 10%（>$100）| $31.03 | 11,640 | $122,355 | 70.6% |

**結論：截尾越多 Surplus 越低。** 雖然少數大額捐款者（$750+）佔比只有 1%，但他們的貢獻是真實的，不應在計算均值時排除。均值是期望值計算的正確統計量。

---

### 簡單一句話

> Level 2 是問：「這個人被聯繫的機率有多高？」
>
> X-Learner 是問：「**這個人因為被聯繫，額外多貢獻了多少個百分點的捐款率？乘上均值後划不划算？**」
>
> 後者的問題更直接對應業務目標。
